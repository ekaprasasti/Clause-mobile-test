Pengenalan

Apa itu Agentic AI?

Memahami paradigma baru kecerdasan buatan yang mampu bertindak secara otonom

Agentic AI adalah sistem kecerdasan buatan yang dirancang untuk bertindak secara otonom dalam mencapai tujuan yang kompleks — bukan hanya merespons pertanyaan, tetapi secara aktif merencanakan, mengambil keputusan, menggunakan alat, dan beradaptasi dengan lingkungan.

Berbeda dengan AI konvensional yang hanya menjawab satu pertanyaan per interaksi, Agentic AI mampu menjalankan rangkaian tindakan berurutan, memonitor hasilnya, dan menyesuaikan strategi secara dinamis untuk menyelesaikan tugas multi-langkah yang kompleks.

AI Konvensional

  • Merespons satu pertanyaan
  • Tidak memiliki memori antar sesi
  • Tidak dapat menggunakan tools
  • Pasif menunggu instruksi
→

Agentic AI

  • Menjalankan tugas multi-langkah
  • Memiliki memori & konteks persisten
  • Menggunakan berbagai tools & API
  • Aktif mengambil inisiatif

Cara Kerja

Loop Kognitif Agentic AI

Siklus inti yang menggerakkan setiap agen AI modern

AGENT Loop Utama
👁️

1. Perceive

Agen mengamati lingkungan, mengumpulkan data dari tools, API, database, dan input pengguna

🧠

2. Reason

Memproses informasi menggunakan LLM untuk memahami situasi dan konteks saat ini

📋

3. Plan

Merancang strategi dan urutan tindakan terbaik untuk mencapai tujuan

⚡

4. Act

Mengeksekusi tindakan: memanggil tools, mengupdate state, berinteraksi dengan sistem

🔍

5. Reflect

Mengevaluasi hasil, belajar dari pengalaman, dan menyesuaikan strategi

Pola Arsitektur Utama

🔗

ReAct Pattern

Reasoning + Acting — agen bergantian berpikir (thought) dan bertindak (action) secara iteratif hingga mencapai jawaban akhir.

Thought: Saya perlu mencari data...
Action: search("data cuaca")
Observation: Suhu 28°C
Thought: Sudah dapat data
Answer: Cuaca hari ini 28°C
🌳

Tree of Thoughts

Eksplorasi multiple jalur pemikiran secara paralel, memilih jalur terbaik berdasarkan evaluasi sistematis.

Root: Masalah kompleks
├── Path A → Evaluasi: 0.8
├── Path B → Evaluasi: 0.5
└── Path C → Evaluasi: 0.9 ✓
🤝

Multi-Agent System

Beberapa agen bekerja sama, masing-masing dengan spesialisasi berbeda, dikoordinasi oleh orchestrator agent.

Orchestrator
├── Research Agent
├── Code Agent
├── Review Agent
└── Deploy Agent
💾

RAG + Memory

Retrieval-Augmented Generation dikombinasikan dengan sistem memori untuk konteks yang kaya dan persisten.

Query → Vector Search
→ Retrieve Context
→ Augment Prompt
→ Generate Response
→ Store Memory

Memory System

Sistem penyimpanan informasi multi-level: working memory (konteks), episodic memory (pengalaman), dan semantic memory (pengetahuan).

Short-term Long-term Vector DB

Planning Engine

Modul yang memecah tujuan kompleks menjadi sub-tugas yang dapat dikelola, membuat rencana eksekusi yang optimal.

Task Decomposition Scheduling
👁️

Perception Module

Mengumpulkan dan memproses input dari berbagai sumber: teks, gambar, data terstruktur, output tools, dan lingkungan.

Multi-modal Data Processing
🔄

Feedback Loop

Mekanisme evaluasi dan pembelajaran: menganalisis hasil, mengidentifikasi kesalahan, dan memperbaiki strategi secara dinamis.

Self-evaluation Adaptation

02

Membuat Agen Pertama

Definisikan dan jalankan agen sederhana dengan beberapa tools.

basic_agent.py 
import asyncio
from openclaw import Agent
from openclaw.tools import WebSearch, Calculator, CodeRunner

# Definisi agen sederhana
agent = Agent(
    name="my_first_agent",
    model="claude-3-5-sonnet-20241022",
    role="""
    Kamu adalah asisten AI yang helpful dan cerdas.
    Gunakan tools yang tersedia untuk membantu
    pengguna menyelesaikan tugas mereka.
    """,
    tools=[
        WebSearch(),
        Calculator(),
        CodeRunner(language="python")
    ],
    max_iterations=15
)

# Jalankan agen
async def main():
    result = await agent.run(
        "Cari informasi terbaru tentang AI, "
        "lalu buat ringkasan dalam 200 kata"
    )
    print(result.output)

asyncio.run(main())
03

Multi-Agent System

Bangun sistem dengan beberapa agen yang bekerja sama secara terkoordinasi.

multi_agent.py 
from openclaw import Agent, Orchestrator
from openclaw.tools import WebSearch, CodeRunner, FileTool

# Agen Spesialis
researcher = Agent(
    name="researcher",
    model="claude-3-5-sonnet-20241022",
    role="Research dan kumpulkan informasi akurat",
    tools=[WebSearch()]
)

coder = Agent(
    name="coder",
    model="claude-3-5-sonnet-20241022",
    role="Tulis kode Python yang clean dan efisien",
    tools=[CodeRunner(), FileTool()]
)

reviewer = Agent(
    name="reviewer",
    model="claude-3-opus-20240229",
    role="Review kode dan berikan feedback detail",
    tools=[CodeRunner()]
)

# Orchestrator mengkoordinasi agen
orchestrator = Orchestrator(
    agents=[researcher, coder, reviewer],
    strategy="sequential",  # atau "parallel"
    max_handoffs=10
)

# Jalankan pipeline
result = await orchestrator.run(
    "Buat REST API sederhana untuk manajemen todo"
)
04

Agen dengan Memori Persisten

Tambahkan kemampuan mengingat informasi lintas sesi menggunakan vector store.

memory_agent.py 
from openclaw import Agent
from openclaw.memory import VectorMemory, EpisodicMemory
from openclaw.memory.backends import ChromaDB

# Setup sistem memori
long_term_memory = VectorMemory(
    backend=ChromaDB(collection="user_data"),
    embedding_model="text-embedding-3-small",
    top_k=5  # Ambil 5 memory paling relevan
)

episodic_memory = EpisodicMemory(
    max_episodes=100,
    summarize_after=20  # Auto-summarize setelah 20 interaksi
)

agent = Agent(
    name="personal_assistant",
    model="claude-3-5-sonnet-20241022",
    role="Asisten personal yang mengingat preferensi user",
    memory=[long_term_memory, episodic_memory],
    tools=["web_search", "calendar", "email"]
)

# Sesi 1: Menyimpan preferensi
await agent.run("Saya lebih suka Python untuk backend")

# Sesi 2 (lain waktu): Agent masih ingat!
response = await agent.run(
    "Rekomendasikan framework web untuk project saya"
)
# → Agent akan merekomendasikan FastAPI/Django (Python)
05

Deploy ke Production

Jadikan agen Anda accessible sebagai REST API atau service yang skalabel.

server.py 
from openclaw.server import AgentServer
from openclaw import Agent

# Definisi agen
agent = Agent(
    name="production_agent",
    model="claude-3-5-sonnet-20241022",
    # ... konfigurasi agen
)

# Bungkus sebagai HTTP server
server = AgentServer(
    agent=agent,
    host="0.0.0.0",
    port=8080,

    # Rate limiting
    rate_limit="100/minute",

    # Auth middleware
    auth="bearer_token",

    # WebSocket untuk streaming
    streaming=True
)

# Run server
# Endpoints otomatis tersedia:
# POST /agent/run       → Jalankan agen
# GET  /agent/status    → Status agen
# WS   /agent/stream    → Streaming output
server.start()

# Atau deploy dengan Docker:
# openclaw deploy --target docker
# openclaw deploy --target kubernetes

Tantangan & Best Practices

Hal-hal penting yang perlu diperhatikan saat membangun sistem Agentic AI

⚠️

Tantangan Utama

  • Hallucination & Reliability LLM dapat menghasilkan informasi yang salah
  • Cost Management Loop yang panjang dapat menghabiskan banyak token
  • Safety & Alignment Agen otonom dapat mengambil tindakan berbahaya
  • Observability Sulit men-debug sistem multi-step yang kompleks
  • Latency Banyak LLM call = response time yang lebih lama
✅

Best Practices

  • Human-in-the-loop Tambahkan checkpoint manusia untuk keputusan kritis
  • Guardrails Definisikan batasan jelas tentang apa yang agen boleh lakukan
  • Iterasi Terbatas Tetapkan max_iterations untuk mencegah infinite loop
  • Logging Komprehensif Catat setiap langkah untuk debugging dan audit
  • Testing & Evals Uji agen secara menyeluruh sebelum production
